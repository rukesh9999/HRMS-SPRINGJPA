What is BhaluV1

Any large data-driven GUI application is built by assembling GUI components (that are dynamically built at run-time by using real-time data) for building larger and larger components and finally the application, which further requires creating communication code for allowing collaboration between the GUI components. Any comprehensive GUI library must allow building any kind of reusable classes for presenting any kind of GUI components and any kind of container components for assembling the GUI components. 7

Each of the reusable Java Classes in Bhalu implements the presentation-logic part to present a GUI component, which requires inputting application data by using simple constructor and set-methods supported by the reusable Java class. The objective of the of Bhalu is to provide a large and ever expanding set of reusable Java classes for generating presentation code for any kind of GUI component. That is, there will be one or more reusable Java classes for presenting one or more kinds of Pie charts, one reusable Java class for presenting each kind of Line-chart, one reusable Java class for presenting each kind of Tree-component, and yet another reusable Java class for presenting each kind of a dial or meter, and so on.

Building a large GUI application requires assembling multiple GUI components and creating communication code for allowing collaboration between the components. To include each of the data driven GUI component needs implementing (i) application logic, which access data-sets at run-time from data sources and process (e.g. iterate over) data-sets for calculating necessary summaries of application data, and (ii) presentation logic to present the GUI component using the application data.
